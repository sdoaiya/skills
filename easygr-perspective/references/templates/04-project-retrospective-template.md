# EasyGR 模板 04｜项目复盘版

> 适用场景：重点项目、活动项目、区域项目、试点项目复盘。  
> 核心要求：**不是复述过程，而是判断这件事卡在哪一环、为什么卡、下一轮怎么改。**

---

## 一、项目基本信息

- 项目名称：
- 项目周期：
- 项目负责人：
- 协同角色：
- 复盘时间：

---

## 二、项目原目标

写清三层目标：

1. 政府目标
2. 企业目标
3. 内部经营目标

---

## 三、实际结果

分开写：

### 过程结果
- 拜访
- 活动
- 邀约
- 培训

### 经营结果
- 名单
- 线索
- 跟进
- 样板
- 签约/订单（如可公开）

### 沉淀结果
- 模板
- SOP
- 案例
- 复盘

---

## 四、按六段闭环复盘

### 1. 开门
- 是否打开了正确入口？
- 入口质量如何？

### 2. 造场
- 场景是否适合转化？
- 是真转化场还是热闹场？

### 3. 邀约
- 名单质量如何？
- 企业分层是否准确？

### 4. 承接
- 销售是否及时接住？
- 线索是否有效流转？

### 5. 汇报
- 是否有阶段成果反馈给政府/管理层？

### 6. 复制
- 是否形成可复制动作？
- 是否具备外溢条件？

---

|---|---|---|---|---|
|  |  |  |  |  |

## 六、下一轮修正动作

按 3 个层面写：

### 1. 立刻要改的
- 本周内完成

### 2. 下轮机制要改的
- 流程
- 分工
- 台账
- 节奏

### 3. 长期沉淀要补的
- 模板
- SOP
- 知识库
- 培训内容

---

## 七、这次沉淀了什么

- 一个模板
- 一个案例
- 一条话术
- 一套台账字段
- 一条风险红线

如果这里写不出来，说明这次还没有真正沉淀为组织能力。
