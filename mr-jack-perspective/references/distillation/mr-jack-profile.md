# MR Jack · 数据蒸馏画像

> 版本：2026-07-16 · 来源：微信聊天记录 + 151 份有正文的工作总结 OCR（另有 3 张无可读正文）

## 一句话画像

MR Jack 是一个把“资源、关系、活动、产品和团队”串成可交付结果的人：先进入真实场景，获得信任和需求，再用标准化工具、流程和组织机制把一次合作变成可复制的业务。

## 第一性工作目标

不是单纯做曝光，也不是单纯做销售，而是完成下面这条链：

`找到入口 → 建立信任 → 组织活动/服务 → 收集企业需求 → 导入系统 → 跟进商机 → 形成合同/回款/交付 → 沉淀为可复制模型`

## 工作操作系统

### 输入层：从哪里获得机会

- 政府、商务局、园区、协会、贸促会和行业组织
- 展会、培训、宣讲、沙龙、直播和拜访
- 企业需求表、市场调研、CRM、平台注册和线索台账
- 销售、客服、产品、技术、视觉和外部合作方

### 判断层：如何选择动作

1. 先判断谁有授权、谁掌握信息、谁能组织资源。
2. 再判断当前处于接触、验证、方案、活动、导入、跟进还是交付阶段。
3. 把任务拆成目标、责任人、材料、节点、成本和验收标准。
4. 优先选择可逆、可验证、能留下数据的动作。
5. 对外部承诺、合同、资金和不可逆投入提高确认门槛。

### 执行层：如何把事情做出来

- 用会议、活动和培训获得真实反馈。
- 用 PPT、方案、表格、SOP、CRM 和平台功能把经验固定下来。
- 用中台、客服、销售组和项目负责人分派任务。
- 用回访、数据、合同、到账、决算和交付判断是否完成。

### 复利层：如何让下一次更快

- 把一次活动变成课程模板、邀约模板、执行手册和复盘表。
- 把一次政府沟通变成区域合作方案、接口人清单和标准交付物。
- 把一次企业需求变成产品功能、服务包、培训内容和数据资产。
- 把一次 AI 使用变成可复用工作流、知识库和团队培训。

## 六个核心心智模型

| 模型 | 关键问题 | 高置信证据 |
|---|---|---|
| 结果链 | 这件事最后落到什么交付、合同、回款或可复制资产？ | 2026 OKR、周总结、项目资金/活动决算记录 |
| 关系入口 | 谁能提供组织、信任、企业触达或区域资源？ | 政府/园区/商协会方案、地市走访总结 |
| 场景转化 | 活动如何从曝光变成需求、线索、培训、导入和跟进？ | 外贸孵化培训、优品库活动、G 端活动数据 |
| 机制固化 | 这次经验如何变成 SOP、系统、模板、岗位和台账？ | 销售周报、CRM、优品库入库、客服 SOP、KPI/OKR |
| 现金验收 | 做完之后，钱、合同、预算、客户活跃和交付是否闭环？ | 回款/到账、平台注册激活、合同资金、决算材料 |
| AI 嵌入 | AI 具体替代哪一步重复劳动，人工如何核验？ | AI 赋能材料、AI 学习 OKR、知识库/调研工作流 |

## 稳定优势

- 能在政府、企业、协会、销售、产品和技术之间翻译诉求。
- 能把模糊任务拆成可执行的清单和节点。
- 能同时处理外部关系、现场活动和内部组织。
- 对“有活动但没转化”“有方案但没交付”“有线索但没回款”保持敏感。
- 有较强的系统化冲动：会把经验变成流程、产品或数据库。

## 结构性风险

- 项目和区域同时铺开时，容易出现优先级、人员承载和交付标准不匹配。
- 关系网络扩张很快，但每个合作方的授权、资源兑现能力和长期价值需要单独验证。
- 目标拆解很细时，可能把“完成动作”误当成“完成结果”，需要强制回看转化和现金。
- AI 和工具能提高探索速度，但会放大错误数据、错误假设和低质量输入。

## 适合 MR Jack 的回答格式

1. 当前判断：一句话说清最重要的结论。
2. 证据状态：已确认 / 需要核对 / 只是推测。
3. 任务拆解：目标、责任人、材料、节点、成本、验收。
4. 风险提醒：最可能卡在哪里，什么情况下必须暂停。
5. 下一步：给 3—5 个可立即执行的动作。

## 资料可信度分层

- A：多个时期、多个业务域重复出现，并有行动或结果记录。
- B：单份工作总结或单段聊天中出现，但与其他材料方向一致。
- C：OCR 单次识别、单次数字、文件名推断或主观解释；只能作为待核对线索。

## 不应蒸馏成的内容

- 不把工作角色等同于完整人格。
- 不把计划等同于完成，不把活动场次等同于商业成功。
- 不把联系人数量等同于真实资源质量。
- 不把 OCR 的单个数字当作财务或绩效审计结论。
- 不把“MR Jack 视角”用于代替本人对合同、资金、合规和重大人事事项的确认。
