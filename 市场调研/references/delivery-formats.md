# Word、PDF、HTML 交付

## 默认交付包

输出目录应包含：

- `full-report.md`：完整 Markdown 源稿。
- `report.html`：静态 HTML 报告。
- `report.pdf`：PDF 正式版。
- `report.docx`：Word 正式版。
- `data/`：原始数据、清洗表、测算表。
- `charts/`：PNG 图表；必要时保留 SVG。
- `sources/`：引用清单、来源登记表、PDF 摘要表。

## 生成顺序

1. 数据表和引用表。
2. 图表资产，优先 PNG，必要时保留 SVG。
3. Markdown 主稿。
4. HTML 排版。
5. 从 HTML 打印 PDF，或使用可靠 PDF 工具生成。
6. Word 版本，插入图表资产、目录、页眉页脚和图注。

## Word 要求

- 封面、目录、页眉页脚、页码、章节编号。
- 中文字体使用微软雅黑/宋体兼容，英文和数字使用清晰无衬线或衬线字体。
- 图表使用独立图号和图注。
- 表格表头深色或浅灰底，正文留足行距。
- 长表放附录，正文只保留关键视图。
- 生成后用文本抽取或打开检查确认内容完整。

## PDF 要求

- 优先由 HTML 或 Word 稳定导出。
- 检查封面、目录、页码、图表清晰度和分页。
- 图表不可模糊；正文 100% 缩放可读。

## HTML 要求

- 单页静态报告优先，便于分享和浏览。
- 使用固定内容宽度、清晰目录、章节锚点、响应式图表。
- 图表资产用 PNG 嵌入，避免依赖外部网络。
- 参考文献可折叠，但核心来源要能在相关段落附近看到。

## 排版风格

- 专业咨询白皮书风格：克制、留白清晰、数据密度高。
- 不使用花哨背景、过量渐变、装饰性大图。
- 每章开头有结论摘要；图表与解释紧邻。
- 附录承担方法、口径、来源和长表，不打断主线阅读。

## 验收清单

- Word、PDF、HTML 三个版本均存在。
- 正文超过用户要求的字数。
- 每个关键数字都有来源。
- 每个图表都有数据源、单位、年份。
- 目录、页码、图号、表号连续。
- 源稿、数据、图表资产可复用。
