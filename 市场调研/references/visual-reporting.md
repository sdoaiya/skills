# 图表与专业表现形式

## 图表资产原则

先做图表资产，再排版报告。每个图表必须有：

- 图表编号
- 支撑的问题
- 一句话结论
- 数据源和证据等级
- 数据口径、年份、单位
- 图表类型
- 放置章节
- 导出格式
- QA 结果

## 推荐图表类型

| 分析问题 | 推荐图表 |
|---|---|
| 贸易额随时间变化 | 折线图、面积图、分面折线 |
| 产品结构变化 | 堆叠柱状图、100% 堆叠柱状图 |
| 国家排名 | 横向条形图、排行榜 |
| 竞争来源国份额 | 堆叠条形图、矩阵表 |
| 市场吸引力比较 | 气泡图、热力图、评分矩阵 |
| 风险与机会 | 2x2 矩阵、雷达图、热力图 |
| 市场规模测算 | 瀑布图、敏感性表、场景表 |
| 国别卡片 | KPI 卡片 + 小趋势图 + 风险标签 |
| 产业链/渠道 | 流程图、价值链图、Sankey 图 |
| 地理分布 | 分级设色地图、区域小倍图 |

## 图表数量建议

- 1 万字报告：12-18 个图表/表格。
- 2-3 万字白皮书：20-35 个图表/表格。
- 4 万字以上深度报告：35-60 个图表/表格。

图表和表格都要服务结论。不要为了显得丰富而堆砌无解释的图。

## 设计规范

- 使用白底或近白底，深色正文，灰色网格线。
- 主色建议：深蓝、钢蓝、金色、橄榄、橙色，避免全篇单一紫蓝或过度渐变。
- 每张图有标题、单位、时间范围、数据来源。
- 长国家名用横向条形图，避免标签挤压。
- 同类图保持同一色义：例如中国为同一强调色，其他来源国为中性色或分组色。
- 重要结论用直接标注，不依赖读者自己看图猜。
- 图表旁边必须有解释段落，说明怎么读、结论是什么、对决策意味着什么。

## 图表 QA

交付前逐项检查：

- 数据粒度与结论一致。
- 坐标轴、单位、年份、币种明确。
- 标签不重叠、不裁切。
- 排名排序合理。
- 同一指标跨图单位一致。
- 正负值、增长率和份额没有混淆。
- 图表源数据可追溯到数据登记表。
- Word/PDF/HTML 中均清晰可读。
